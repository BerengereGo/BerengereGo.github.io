Principes de base

On va commencer par poser un fondement du CSS: la notion de boite ou Box. Ceci veut tout simplement dire que l'on va imbriquer des morceaux de code HTML dans des "boites" afin d'appliquer des styles dessus. Voici comment est représentée une Boite : . Plusieurs parties sont à distinguer :

  • en bleu = taille réelle de la boite
  • padding = espace entre la boite et la bordure
  • border = bordure
  • margin = espace extérieur à la bordure

Notons que chaque parties est paramétrable. Ainsi, on aura par exemple margin : 1px 2px 3px 4px ; équivalent à on ajoute 1px (px = pixel) en haut, 2px à droite, 3px en bas et 4px à gauche de la bordure. De même, on aura padding : 4px 3px 2px 1px pour on ajoute 4px en haut, 3px à droite, 2px en bas et 1px ) gauche de l'élément en bleu. Par convention, on nomme d'abord le haut puis on tourne dans le sens des aiguilles d'une montre. Cependant, on peut également ne paramètrer qu'un seul des 4 paramètres, on le désignera alors spécifiquement : margin-top, margin-bottom, margin-left ou margin-right, idem pour padding. Mais on reverra tout ça ultérieurement.

Il existe deux balises de bases :

  • <div> ... </div>. Ceci est une boite de type BLOCK. Elle sert juste de conteneur à ce qui se trouve dedans. Elle crée simplement un nouveau bloc sans marges ni enrobement (padding)
  • <span> ... </span> Equivaut à div mais de typer LIGNE. Seule, elle n'a aucun effet sur le rendu du code html. Cependant, elle devient très utile lorsqu'on lui attribut des styles

Voici des exemples d'autres balises HTML qui s'affichent naturellement en block : < titre > ; < p > ; < ol > ; < table > ... Voici d'autres exemples de balises HTML qui s'affichent naturellement en inline : < a > ; < table >.

L'objectif ultime est d'ajouter un minimun de code pour un maximum d'effet, sinon on a toujours le risque d'affecter sa maintenabilité.

Avant d'attaquer, voici comment on déclare le lien entre le doument HTML et le document CSS :

 
        <head>
          <link rel = "stylesheet" type = "text/css" href = "votredocument.css" />
        </head>

Un peu d'explications. Déja, le lien se stipule dans le <head>. Rappelons-nous que c'est LA zone réservée aux méta-données (avec le titre du document et utf-8). Petite remarque en passant, la balise <link/>> est auto-fermante. Ensuite, on a besoin de 3 attributs:

  • rel : indique la nature du lien, la valeur de l'attribut est donc "stylesheet"
  • type : ici "text/css"
  • href : indique où se trouve l'URL de mon fichier.css

Schématiquement, une feuille de styles ressemble à ça: 

        balise1 {
          propriete1: valeur1;
          propriete2: valeur2;
          propriete3: valeur3;
          }

        balise2 {
          propriete1: valeur1;
          propriete2: valeur2;
          propriete3: valeur3;
          propriete4: valeur4;
          }

        balise3 {
          propriete1: valeur1;
          }

Le display

La propriété CSS display est la plus importante pour contrôler la mise en page. Chaque élément à une valeur display par défaut dépendante du type de l'élément. Les valeurs par défaut de la plupart des éléments sont block ou inline.

Les classes et les id

L'utilisation des attributs classe et id permet de créer des ensembles auxquels on appliquera les mêmes paramètres. Il y a peu de différences entre ces 2 attributs, excepté que l'id ne sera utilsable qu'une et seule fois! Il y a cependant des règles de priorités sur l'éxécution du css :

  • propriétés id > propriétés classe
  • propriétés classe > propriétés élément

Voici un exemple d'id et de classe ;

 <header class = "header">
              <img id  = "tortue" src = "tortue.png"
                                     alt = "Bien le bonjour de la tortue rigolote" > 
              <h1 id = "site">  HTML pour les Zéros? Par ici ! </h1>
            </header>

Ici, on a 3 boites. Une boite classe qui a pour valeur "header", et 2 boites id qui ont pour valeurs "tortue" et "site". On pourra appliquer des propriétés générales dans la classe qui affecteront les enfants de cette classe, ici les id (boites dans la boite) puis appliquer des propriétés relatives à chaque id spécifiquement.

Les sélecteurs

Pour rappel, un sélecteur permet d'appeler des éléments en HTML afin de leur attribuer des propriétés. Nous avons déjà vu 3 types de sélecteurs qui sont l'élément ( ), la classe (.) et l'id (#). A cette liste on peut ajouter plusieurs. Sélecteur de voisins immédiats + . Ex dd + dd : la règle s'applique à la définition suivie d'une définition : 

 
        dd + dd {
          <dl> 
            <dt> terme1 </dt> 
            <dd> définition </dd>
            <dd> définition2 </dd>
          </dl> 
          }

Sélecteur de voisins indirects ~ . Ex img ~ a : Les régles s'appliquent à l'image si elle est suivie d'une référence, même s'il y a d'autres éléments entre.

 
        img ~ a {
          <img> img </img>
          <div> </div>
          <p> ... </p>
          <a img  src = "URL de l'image"
                     alt = "texte à afficher si l'image ne s'affiche pas" > </a>
        }

Sélecteur d'un parent direct (où plutôt de l'enfant) > . Ex div > p : la règle s'applique à l'enfant p de div s'il le suit directement.

        div > p {
        <div>
        <p>
        ...
        </div> 
        }

Sélecteur parent indirect , même chose mais il peut y avoir des enfants entre div et p (div p).

Sélecteur universel * . Il sélectionne toutes les balises, sans exception.

sélecteur pour sélectionner un élément en fonction de la valeur de son attribut [] . Ex [class = "rouge"] : La règle s'applique à tout les éléments entre accolades ayant pour valeur "rouge"

Les positions

Pour attaquer les positions, il faut comprendre la notion ultra-complexe de flux... ordre dans lequel s'affiche les éléments dans la page! Pas si complexe en fait. Juste, ne pas oublier que ce sera soit des éléments en block (les uns en-dessous des autres) ou bien des éléments en ligne (les uns à la suite des autres sur la ligne). 3 possibilités:

  • static : valeur par défaut du positionnement
  • absolute : on retire l'élément du flux, sa position se fait donc par rapport à son conteneur
  • relative : on définit la position d'un élément d'un flux à partir de sa position initiale qu'il aurait dû avoir
  • fixed : l'élément sera positionné et ne bougera plus lors du défilment de la page
